8. Modelo de Datos

El sistema se compone de un modelo relacional orientado a trazabilidad y control institucional de eventos.

Entidades Principales

  • users – Registro de funcionarios y administradores.
  • roles – Definición de perfiles de acceso.
  • users_roles – Asociación entre usuarios y roles.
  • subscribes – Períodos habilitados de teletrabajo.
  • registers – Eventos originales generados por el funcionario.
  • registers_modifieds – Eventos administrativos adicionales vinculados a un registro base.
rule
Los eventos administrativos no reemplazan ni eliminan registros originales. Se almacenan como eventos adicionales con vínculo al registro base correspondiente.
info
Todas las entidades incorporan campos de auditoría (created_at, updated_at, deleted_at), asegurando trazabilidad completa.
