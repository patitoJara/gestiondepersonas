10. Gestión de Sesión de Usuario

timer
El sistema incorpora un mecanismo de control de sesión activa con el objetivo de mantener la seguridad, trazabilidad y continuidad operativa del sistema.

Visualización en Interfaz:
El estado de la sesión se presenta en la barra superior del sistema (Topbar), permitiendo al usuario visualizar en tiempo real su estado de conexión.

Características:
- El sistema mantiene una sesión activa por usuario autenticado.
- Se visualiza un temporizador de sesión en la interfaz superior.
- El temporizador indica el tiempo restante antes del cierre automático de sesión.
- El sistema presenta estados visuales de advertencia y criticidad según el tiempo restante.

Estados del temporizador:
- Normal: funcionamiento regular.
- Advertencia: cuando el tiempo restante es reducido.
- Crítico: cuando la sesión está próxima a expirar.

Extensión de sesión:
- El usuario puede extender manualmente su sesión mediante el botón "Extender".
- La extensión se encuentra sujeta a validaciones del backend.

Cierre de sesión:
- Si la sesión expira, el sistema cerrará automáticamente la sesión del usuario.
- El usuario deberá autenticarse nuevamente para continuar utilizando el sistema.

Este mecanismo permite reducir riesgos de uso indeb en sesiones prolongadas o desatendidas.
