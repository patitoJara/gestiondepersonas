Módulo de Transferencia de Demandante entre Programas

info
El módulo de Transferencia permite trasladar formalmente un demandante desde un programa activo a otro dentro del sistema RDA-SM, asegurando trazabilidad clínica, notificación institucional y consistencia normativa.

Objetivo del módulo

La transferencia está diseñada para mantener la continuidad del proceso terapéutico cuando un usuario requiere cambiar de programa por razones clínicas, administrativas o territoriales.

verified
La transferencia no elimina información previa. El registro clínico original mantiene su historial, y el nuevo programa asume la continuidad del proceso.

Flujo del proceso de transferencia

  • Paso 1 – Buscar RUT: se ingresa el RUT del demandante. El sistema valida formato y existencia de registros asociados.
  • Paso 2 – Seleccionar Registro: si existen múltiples registros, el usuario selecciona el correspondiente al programa activo.
  • Paso 3 – Revisión Clínica: se muestra una ficha resumen tipo hoja clínica con datos del postulante y del registro actual.
  • Paso 4 – Programa Destino: se selecciona el programa al cual será transferido el demandante. El sistema excluye automáticamente el programa de origen.
  • Paso 5 – Confirmación: se muestra un resumen de la transferencia incluyendo programa destino y lista de correos que serán notificados.
  • Paso 6 – Ejecución: el sistema confirma la operación y registra la transferencia.

Notificación automática

mail
Al ejecutar la transferencia, el sistema genera una notificación por correo electrónico a:
  • Profesionales asociados al programa de origen
  • Profesionales asociados al programa destino
  • Usuarios con rol Supervisor
  • Usuarios con rol Administrador

Esta notificación informa que el demandante ha sido transferido, permitiendo coordinación clínica inmediata entre equipos.

Validaciones del sistema

warning
El sistema no permite ejecutar la transferencia si:
  • No se ha seleccionado un programa destino.
  • El usuario no posee permisos suficientes.
  • El registro clínico no está vigente.

Seguridad y control

lock
El proceso mantiene trazabilidad institucional, registrando:
  • Usuario que ejecuta la transferencia
  • Programa de origen
  • Programa destino
  • Fecha y hora del evento

Finalización del proceso

Una vez completada la transferencia, el sistema muestra un mensaje de confirmación y permite reiniciar el módulo para realizar una nueva transferencia si fuese necesario.

check_circle
La transferencia es una acción formal dentro del sistema y debe ser realizada únicamente cuando exista validación clínica o administrativa correspondiente.
