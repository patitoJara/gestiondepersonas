Módulo de Demandas

info
El módulo de Demandas permite registrar, gestionar y dar seguimiento a las solicitudes de atención en Salud Mental y Adicciones, asegurando trazabilidad clínica, control normativo y consistencia de la información.

Tipos de operación disponibles

  • Nueva Demanda: permite crear un registro completo desde cero. El sistema valida RUT, sustancia principal y datos clínicos obligatorios, aplicando valores por defecto según normativa.
  • Editar Demanda: permite modificar una demanda existente del mismo programa activo. Los datos del postulante y del referente quedan bloqueados para resguardar la integridad clínica.
  • Usar datos (mismo programa): crea una nueva demanda reutilizando información previa del postulante y referente, generando un registro clínico independiente.
  • Usar datos (otro programa): permite generar una nueva demanda asociada al programa activo, reutilizando datos base de otro programa.
search
El RUT es el punto de inicio del proceso. Al ingresar un RUT válido, el sistema busca demandas previas y permite editar una existente, reutilizar datos o crear una nueva demanda, evitando duplicidades.

Gestión de citaciones

Cada demanda puede contener múltiples citaciones, donde se registra la fecha, hora, profesión, profesional y estado de atención.

  • Agendado
  • Se presentó
  • No se presentó
  • Cancela programa
warning
El sistema aplica reglas clínicas automáticas según la previsión de salud del postulante. En casos no FONASA, la demanda queda marcada como No aceptada, pudiendo continuar el registro bajo responsabilidad del profesional.

Sustancias asociadas

Es obligatorio seleccionar una sustancia principal. El sistema permite agregar sustancias secundarias, asegurando que no existan duplicidades entre ellas.

lock
Antes de guardar o actualizar una demanda, el sistema valida la existencia de un RUT válido, la selección de una sustancia principal, el cumplimiento de campos clínicos obligatorios y la vigencia de la sesión del usuario.

Control de salida con cambios sin guardar

warning
Cuando una demanda presenta modificaciones sin guardar, el sistema activa mecanismos de protección para evitar la pérdida accidental de información.

El sistema identifica que existen cambios pendientes cuando el botón Guardar Demanda se encuentra habilitado, lo que indica que el formulario contiene modificaciones pendientes de guardar respecto del último estado almacenado.

  • Navegación interna: si el usuario intenta salir del módulo de Demandas (por menú, cambio de vista o retorno a Inicio), el sistema mostrará un mensaje de confirmación propio del sistema indicando que existen modificaciones sin guardar, permitiendo cancelar la acción o salir perdiendo los cambios.
  • Recarga del navegador (F5) o cierre de pestaña: el navegador mostrará una advertencia estándar de seguridad del navegador. En navegadores como Google Chrome y Microsoft Edge, el mensaje mostrado es similar a:

    "Changes you made may not be saved."

    Este mensaje es generado por el navegador y no puede ser personalizado ni traducido por el sistema.
info
Este control se aplica únicamente cuando existen cambios pendientes y no interfiere con la navegación normal cuando la demanda se encuentra guardada.
