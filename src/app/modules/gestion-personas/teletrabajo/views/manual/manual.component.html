2. Alcance General de la Plataforma

La plataforma Gestión de las Personas centraliza distintas funcionalidades institucionales asociadas a la gestión operativa de funcionarios y funcionarias del Servicio de Salud Magallanes.

Entre sus principales módulos se encuentran Teletrabajo, Beneficios, Postulación Apoyo Estudios Superiores, Supervisión de Postulaciones, Administración de Usuarios, Administración de Jefaturas y Auditoría Grupos/Jefaturas.

Las opciones visibles en el menú lateral pueden variar según el rol activo del usuario.

5. Módulo Teletrabajo

El módulo Teletrabajo permite registrar eventos de ingreso y salida, consultar registros personales, revisar suscripciones autorizadas y generar reportes de apoyo para supervisión y auditoría.

Las marcaciones se validan con la hora del servidor institucional y con los rangos horarios autorizados para cada jornada.

10. Módulo Beneficios

El módulo Beneficios permite acceder a procesos institucionales de apoyo o postulación definidos por el Servicio de Salud Magallanes.

Actualmente incorpora la opción Postulación Estudios, correspondiente al proceso de postulación al beneficio de Apoyo para Estudios Superiores.

assignment

11. Postulación Apoyo Estudios Superiores

Esta funcionalidad permite al funcionario o funcionaria ingresar una solicitud de postulación, completando antecedentes personales, familiares, académicos, económicos, de salud, vivienda y documentación de respaldo.

El formulario se organiza en etapas sucesivas para facilitar el ingreso ordenado de la información.

  • Datos del afiliado.
  • Grupo familiar.
  • Postulante.
  • Antecedentes académicos.
  • Verificación académica.
  • Ingresos familiares.
  • Gastos.
  • Salud y vivienda.
  • Documentos.
  • Confirmación.
  • Finalizado.

12. Carga de Documentos de Postulación

La etapa de documentos permite adjuntar los antecedentes obligatorios y adicionales requeridos para la evaluación de la postulación.

El sistema informa visualmente el avance de carga, los documentos obligatorios cargados y los documentos pendientes. El usuario puede revisar los archivos antes de enviar la postulación y quitar documentos cuando sea necesario corregir una carga.

13. Supervisión de Postulaciones Bienestar

La supervisión de postulaciones permite revisar solicitudes ingresadas por los funcionarios, consultar antecedentes, visualizar documentos, generar reportes y apoyar el proceso de seguimiento institucional.

Esta funcionalidad está disponible para los perfiles autorizados según los roles definidos en el sistema.

groups

15. Administración de Jefaturas y Grupos

La administración de jefaturas y grupos permite gestionar la relación entre funcionarios, grupos de trabajo y jefaturas responsables.

Desde esta funcionalidad es posible crear nuevas jefaturas, administrar jefaturas existentes, agregar funcionarios, quitar funcionarios y mantener organizada la información para efectos operativos y de supervisión.

Un funcionario puede estar asociado a más de una jefatura cuando exista una necesidad administrativa, como reemplazos, subrogancias, licencias médicas, vacaciones o apoyo temporal entre unidades.

manage_search

16. Auditoría Grupos/Jefaturas

La auditoría de grupos y jefaturas permite revisar la información registrada sin modificarla. Está orientada a consulta, control y respaldo institucional.

Permite visualizar grupos existentes, jefatura responsable, ID del grupo, cantidad de funcionarios asignados, funcionarios pertenecientes a cada grupo, grupos sin funcionarios y grupos con nombres genéricos.

Una jefatura puede existir aunque no tenga funcionarios asignados. En ese caso, el sistema la mantiene visible como Sin funcionarios asignados, permitiendo conservar trazabilidad.

21. Actualización de Versión del Sistema

El sistema Gestión de las Personas puede recibir actualizaciones periódicas de frontend para incorporar mejoras visuales, ajustes de formularios, correcciones de funcionamiento o nuevas funcionalidades.

En algunos casos, el navegador del usuario puede mantener temporalmente una versión anterior de la plataforma. Esto puede provocar que se visualicen pantallas antiguas, botones que no aparecen correctamente, información desactualizada o formularios con datos temporales de una sesión anterior.

Si el usuario visualiza una versión antigua del sistema o detecta que una pantalla no se muestra correctamente, se recomienda cerrar sesión, volver a ingresar y, si el problema continúa, presionar Ctrl + F5 en la pantalla de acceso.

22. Soporte y Recomendaciones Generales

Ante inconvenientes de acceso, visualización, formularios, marcaciones, suscripciones o reportes, el usuario debe comunicarse con el área correspondiente según el módulo utilizado.

Para facilitar la revisión, se recomienda indicar nombre completo, correo institucional, módulo donde ocurre el problema, breve descripción y captura de pantalla cuando sea posible.
