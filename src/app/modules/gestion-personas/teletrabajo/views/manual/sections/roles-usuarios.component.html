3 Modelo de Roles y Asignación de Usuarios

El sistema define la disponibilidad y pertenencia de los usuarios a grupos en base a la relación usuario–grupo y a los roles asignados. Esta lógica permite mantener una estructura clara, consistente y controlada para la gestión del personal.

info La pertenencia a un grupo es determinada únicamente por la relación usuario–grupo. Los roles no determinan pertenencia, pero sí definen el comportamiento y disponibilidad del usuario dentro del sistema.

Reglas de Funcionamiento

Se establecen las siguientes reglas para determinar la disponibilidad de los usuarios:

  • Un usuario puede tener uno o múltiples roles.
  • Un usuario pertenece a un grupo solo si existe una relación usuario–grupo.
  • La disponibilidad de un usuario depende exclusivamente del rol ADMINISTRATIVO.
  • Si un usuario posee múltiples roles, el rol ADMINISTRATIVO se considera dominante para efectos de disponibilidad.
  • Si un usuario no tiene el rol ADMINISTRATIVO, nunca será considerado disponible, independiente de su estado o pertenencia.
  • Un usuario disponible es aquel que tiene rol ADMINISTRATIVO y no pertenece a ningún grupo.
  • Un usuario que cumple funciones de SUPERVISOR o JEFATURA no puede desempeñarse como administrativo dentro del grupo que administra.
